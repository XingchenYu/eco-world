# EcoWorld 玩法循环说明

本文档说明当前 `v4` 设计下，这个项目最终准备做成什么样的游戏、玩家在玩什么，以及当前已经具备和仍然缺失的环节。

## 游戏定位

当前设计下，`EcoWorld` 更接近一种：

- `生态战略游戏`：玩家不是单体角色，而是站在世界层做判断。
- `区域推进游戏`：玩家需要在多个生态区之间选择推进方向。
- `生态系统模拟游戏`：区域不会静止不动，物种、链路、繁荣和风险会持续变化。

它不是：

- 纯动作游戏
- 传统 JRPG 回合战斗
- 纯数值放置游戏
- 纯后台观察器

更准确地说，玩家扮演的是一个 `世界生态总控者`，在世界地图上观察、筛选、比较、锁定并确认不同区域的推进方案，再根据生态反馈不断调整长期战略。

## 玩家在玩什么

当前主设计里，玩家的核心行为不是“操作一个角色移动”，而是反复完成下面这套循环：

1. `读取世界`
   查看世界地图上的区域状态、主链焦点、风险、繁荣和前线网络。

2. `选择战区`
   决定当前关注哪个区域，以及要从哪个前线走廊往外推进。

3. `比较候选分支`
   比较不同落点、不同路线、不同阶段的风险和繁荣，不盲目推进。

4. `切换战略姿态`
   在稳态、均衡、突进、高风险等不同策略视角之间切换。

5. `锁定并确认方案`
   把临时比较结果推进成正式的战区结论，而不是永远停留在观察阶段。

6. `观察生态后果`
   看物种、链路、资源、压力和长期相位如何响应当前决策。

7. `再次调整`
   如果当前推进方向不再优，就改筛选、改路线、改编成、改指令、改定案。

## 三层玩法结构

### 1. 世界层

这是当前已经最明确的一层，也是 Godot 前端正在实现的主界面。

玩家在这一层主要做：

- 切换区域
- 查看世界播报
- 看区域之间的前线连接
- 看二级分支网络
- 比较多个候选落点
- 切换战区模式、推进阶段、筛选模式
- 切换编成、预案、指令
- 锁定定案并确认推进结论

这一层解决的问题是：

- 现在世界上哪里最值得推进
- 哪里风险过高
- 哪条路线更稳
- 当前应该用什么战略姿态

### 2. 区域层

这是下一阶段要继续做强的层。

玩家进入某个区域后，应该能更细地看：

- 当前区域的健康、资源、风险
- 社会相位
- 草原链、尸体资源链、湿地主链
- 核心物种状态
- 领地、捕食、竞争、共生压力

这一层未来不应该只是“看面板”，而要开始支持更明确的区域级决策，例如：

- 重点保育
- 风险缓解
- 资源恢复
- 物种保护或通道修复

### 3. 长期层

这一层是项目的独特性来源。

玩家不是打一场就结束，而是持续跨周期观察：

- 当前决策是否真的改善了繁荣
- 当前策略是否只是短期压住风险
- 哪些区域正在进入长期衰退
- 哪些生态链正在积累恢复窗口
- 哪些世界级压力正在跨区域传导

也就是说，玩家玩的不是一次点击，而是 `长期治理`。

## 当前 Godot 世界入口对应的玩法含义

现在 Godot 世界地图里已经逐步形成了下面这些操作层，它们不是单纯的 UI 名词，而是具体玩法动作：

- `战区推进模式`
  表示玩家先决定当前世界观察的主战区视角。

- `推进阶段`
  表示玩家决定当前是看主走廊阶段，还是看第二阶段分支。

- `落点网络`
  表示玩家不是只看一个目标，而是比较多个可能落点。

- `筛选模式`
  表示玩家按不同战略目标看世界：
  - 综合
  - 稳态
  - 高繁荣
  - 高风险

- `路线调度`
  表示玩家决定主执行、辅执行、回退三种路线关系。

- `编成 / 预案 / 指令`
  表示玩家从“区域推进形态”进一步进入正式的战略控制层。

- `定案 / 锁定 / 确认`
  表示玩家把观察结果推进成真正的战区结论。

这套东西连起来之后，世界层已经不是“看地图”，而是“做战略”。

## 当前已经具备的玩法基础

按现在仓库里的主线，已经比较明确的部分有：

- 活的世界状态，而不是静态关卡
- 多区域并行
- 区域之间存在真实连接和分支网络
- 区域内有持续变化的生态链
- 世界层已经有观察、筛选、比较、锁定、确认这一整套决策结构
- 长期压力、长期记忆、长期相位已经和区域状态联通

所以项目已经越过了“只是展示模拟结果”的阶段，正在进入“把模拟世界变成可玩的战略界面”。

## 当前还缺什么，才算真正可玩

虽然玩法骨架已经很清楚，但离“真正可玩”还差几块硬缺口：

### 1. 区域级可执行动作

现在世界层已经能做很多战略选择，但玩家确认之后，还缺少更明确的区域级动作回报。

后面需要补：

- 具体治理动作
- 行动后果摘要
- 动作对生态链的回灌反馈

### 2. 目标系统

现在已经有很多决策层，但还缺少“为什么现在要这样选”的正式目标。

后面需要补：

- 章节目标
- 区域治理目标
- 长期世界目标
- 失败或退化条件

### 3. 结果可视化

现在已经有很多指标，但还缺少更强的“行动后变化”表达。

后面需要补：

- 决策前后对比
- 关键物种变动提示
- 风险抬升/回落提示
- 区域繁荣或崩溃趋势提示

### 4. 区域下钻玩法

现在世界层越来越像游戏了，但区域层还没有形成真正的“第二主界面”。

后面需要补：

- 区域详情页
- 区域行动页
- 区域图鉴页
- 区域事件页

## 一局游戏的大致体验

如果按当前设计继续推进，未来一局的体验大致会是：

1. 进入世界地图
2. 读取世界当前危险点和繁荣区
3. 选择一个战区作为本轮重点
4. 比较多个落点和推进路线
5. 选编成、选预案、选指令
6. 锁定并确认战区定案
7. 进入区域层观察结果和执行区域级动作
8. 经过若干轮世界更新后，重新回到世界层调整战略

所以它的节奏更接近：

- `世界判断`
- `区域治理`
- `长期反馈`
- `再判断`

而不是单线推进式关卡游戏。

## 当前最准确的玩法结论

一句话总结当前设计：

`EcoWorld` 正在形成的是一个“以世界地图为入口、以区域治理为中层、以长期生态反馈为核心”的生态战略游戏。

当前已经比较完整的是：

- 世界层战略入口
- 多区域比较与推进结构
- 长期生态反馈骨架

当前还需要继续补的是：

- 区域层可执行玩法
- 正式目标系统
- 行动后果表达
- 更完整的区域详情与下钻交互
