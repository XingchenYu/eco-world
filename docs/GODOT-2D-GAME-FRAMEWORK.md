# EcoWorld 2D 底层玩法框架

本文档定义 2D 主线下的底层玩法框架。它不是 UI 草图，而是后续所有系统、数值、任务和区域内容必须遵守的基础规则。

## 1. 产品定义

`EcoWorld` 的 2D 主线不是动作闯关，也不是纯模拟观测器。

它的核心类型是：

- `生态调查冒险`
- `区域撤离循环`
- `后端生态驱动的系统游戏`

玩家扮演的不是“猎人”或“战士”，而是一个进入前线生态区的 `调查员 / 记录者`。

玩家的目标不是杀怪，而是：

- 进入区域
- 判断局势
- 记录关键物种与热点
- 见证生态压力
- 决定何时撤离
- 把情报带回世界层

## 2. 核心体验

一轮区域探索必须同时成立 4 件事：

1. `看得懂`
   玩家能快速读懂地图结构、热点、危险和当前目标。

2. `有取舍`
   玩家不能什么都做完，必须在风险、收益和时间里做选择。

3. `有生态味`
   玩家看到的是生态系统里的压力、竞争、迁移和聚群，不是随机刷怪。

4. `能撤离`
   每次进入区域都必须有“带着情报离开”的闭环，而不是无限停留。

## 3. 单次 Expedition 基础循环

2D 主线的最小可玩循环是：

1. `进入区域`
2. `追踪目标`
3. `主动调查`
4. `承受压力`
5. `判断撤离`
6. `切区或回世界图`

把它拆开后：

### 3.1 进入区域

玩家在入口落地后，需要立刻知道三件事：

- 当前区域主生境是什么
- 当前值得先去的热点是什么
- 当前最大的风险来自哪里

### 3.2 追踪目标

玩家在区域内移动，靠近：

- 动物群
- 生态热点
- 出口

但“靠近”本身不是奖励，只是获得行动资格。

### 3.3 主动调查

当前正式基线是：

- 靠近目标后按住 `Space`
- 对动物执行 `记录`
- 对热点执行 `采样`

也就是说，调查是动作，不是被动触发。

### 3.4 承受压力

玩家在调查过程中，会同时面对：

- 掠食压力
- 追逐爆发
- 危险上升
- 错失观察窗口

这保证探索不是散步。

### 3.5 判断撤离

当玩家收集到足够情报，或者风险过高时，需要决定：

- 继续深入
- 当前撤离
- 直接切去下一区域

### 3.6 回灌世界层

撤离不是结算动画，而是把本轮结果回灌给世界层：

- 发现了什么物种
- 完成了哪些热点观察
- 见证了什么生态事件
- 带回了多少情报

## 4. 三个核心资源

2D 主线所有玩法都围绕这三个资源运转：

### 4.1 情报

情报是玩家的主收益。

来源：

- 首次记录物种
- 完成热点采样
- 见证压力事件
- 见证追猎结果
- 安全撤离

用途：

- 解锁图鉴
- 推进区域阶段
- 支撑世界层决策
- 打开新的区域路线说明

### 4.2 风险

风险是玩家的主成本。

来源：

- 靠近掠食者
- 停留在高压力热点
- 追猎爆发附近逗留
- 带着高价值观察结果继续贪心推进

风险不一定直接导致死亡。
它优先影响：

- 调查稳定度
- 撤离压力
- HUD 提示
- 世界层对本轮 expedition 的评价

### 4.3 观察窗口

观察窗口是时间和空间上的“机会”。

例子：

- 草食群聚到迁徙廊道
- 掠食者开始压迫
- 食腐鸟开始聚集
- 水域动物靠近水源边

玩家玩的不是无限重复交互，而是抓窗口。

## 5. 玩家动作集合

底层动作只保留少数几种：

- `移动`
- `冲刺`
- `调查`
- `撤离`
- `切区`
- `查看图鉴 / 日志`

不要继续增加复杂按键战斗系统。

设计原则：

- 操作必须少
- 决策必须多

## 6. 区域内对象类型

区域内只允许 4 类主对象驱动玩法：

### 6.1 动物群

功能：

- 提供物种记录
- 提供行为观察
- 提供风险读数

### 6.2 热点

功能：

- 提供采样任务
- 指向当前区域真正值得去的地方
- 绑定物种类别要求

### 6.3 出口

功能：

- 提供撤离判断
- 提供切区入口
- 把本轮 expedition 结果送回世界层

### 6.4 生态事件

功能：

- 让区域活起来
- 改变风险和窗口
- 构成日志里的“本轮故事”

## 7. 区域三阶段状态机

所有区域探索都应该落进同一个简化状态机：

### 阶段 A：追踪

目标：

- 找到本轮值得调查的对象

触发条件：

- 进入区域
- 尚未积累足够情报

主要体验：

- 读图
- 靠近
- 判断先去哪

### 阶段 B：记录

目标：

- 完成 1 到 2 个有效调查

触发条件：

- 已接触到动物群或热点

主要体验：

- 按住 `Space`
- 在压力下完成调查
- 决定要不要继续贪更多收益

### 阶段 C：撤离

目标：

- 带着收益退出，而不是死磨全图

触发条件：

- 情报达到最低阈值
- 或危险达到高位

主要体验：

- 值不值得继续
- 该从哪个口走
- 要不要立刻回世界层

## 8. 世界层和区域层怎么接

区域层不是独立小游戏，它是世界层的执行端。

世界层负责：

- 选区
- 选路线
- 看长期趋势

区域层负责：

- 收集实地情报
- 验证世界层判断
- 带回结果

因此区域层的所有主要产出，最终都要可回灌：

- `species discoveries`
- `hotspot surveys`
- `pressure witnessed`
- `chase outcome witnessed`
- `expedition intel score`

## 9. HUD 信息架构

HUD 只保留最必要的 4 块：

### 左上：区域状态

- 区域名
- 当前阶段
- 情报值
- 危险度

### 中上：当前事件

- 当前最重要生态事件
- 当前出口判断
- 当前观察机会

### 左下：当前行动卡

- 当前调查目标
- 调查进度
- 出口说明

### 右侧：本轮任务

- 世界图带入的任务
- 本轮完成条件
- 撤离后对后端生态系统的影响
- 少量代表物种和已记录进度

不再回到屏幕上堆大量长段说明。

当前右侧面板不是资料库。资料库可以以后放到暂停菜单或独立图鉴页，主界面只服务本轮决策。

## 10. 后端数据契约

2D 主线必须直接吃后端世界数据，不另造一套手工关卡逻辑。

最低要求：

- `region_details`
- `dominant_biomes`
- `species_manifest`
- `exploration_hotspots`
- `frontier_links`
- `health_state`

扩展要求：

- `pressure events`
- `behavior anchors`
- `narrative / bulletin`
- `species role tags`

## 11. 不做什么

底层框架明确不做这些：

- 即时战斗
- 技能树
- 装备系统
- 采集制造
- 大量数值 RPG 成长
- 假开放世界跑图

这些都会稀释主循环。

## 12. 当前实现优先级

按顺序做：

1. `调查闭环`
   - 动物记录
   - 热点采样

2. `撤离闭环`
   - 风险
   - 情报
   - 撤离判断

3. `世界回灌`
   - 区域结果进入世界图

4. `区域差异`
   - 不同生境不同玩法压力

5. `长期目标`
   - 不只是单局 expedition

## 13. 当前视觉基线

### 13.1 世界地图

世界地图必须像地图，不像后台仪表盘。

当前基线：

- 使用真实地形栅格底图。
- 地区只保留柔和色晕、地名、小型可点击标记。
- 不默认显示大面积连线网络。
- HUD 只解释当前区域、推荐行动和完整玩法闭环。

禁止回退：

- 大面积矩形卡片铺在地图上。
- 粗重线路覆盖真实地形。
- 中英文混杂的系统字段直接上屏。

### 13.2 区域探索地图

区域探索必须像生态现场，不像积木编辑器。

当前基线：

- 草地、林地、水体、山脊使用自然斑块和低透明纹理。
- 不使用大块圆角矩形当地形。
- 不使用棕色墙式多边形当地貌。
- 迁徙路线不常驻显示；只有捕食、追逐等事件才出现临时反馈线。
- 动物使用 2D 生物剪影，不再用纯圆点代表。

### 13.3 主界面信息量

主界面必须遵守“一眼目标”原则。

玩家进入区域后，3 秒内必须能看懂：

- 当前区域是什么。
- 本轮任务是什么。
- 需要怎么完成。
- 完成后会怎样影响世界。

长日志、完整图鉴、系统细节不能抢占主界面。

## 14. 一句话标准

如果后面任何新增功能不能强化下面这句话，就不该进主线：

`玩家进入一个活生态区域，在风险中主动调查，带着有价值的情报撤离，并把结果反馈回整个世界。`
