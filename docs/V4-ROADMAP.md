# V4 路线图

## 1. 目标

在不再做大重构的前提下，把当前项目升级为完整虚拟生态世界。

## 2. 路线阶段

### Phase A：骨架

- 新建 `world / ecology / data / sim / ui`
- 定义核心数据结构
- 定义 `WorldMap / Region / BiomePatch / HabitatPatch`
- 定义模板和关系表 schema

### Phase B：单区域升级

- 把现有 `Ecosystem` 抽象成 `RegionSimulation`
- 接入模板和关系表
- 保持当前可运行能力

### Phase C：六区世界

- 实现 6 个核心生态区
- 实现区域间资源和迁徙交换
- 新增全球入口视图

### Phase D：关键种机制

- 大象
- 河马
- 鳄鱼
- 狮
- 狼
- 河狸
- 珊瑚
- 清洁鱼

做出真正的局部级联反应。

### Phase E：内容扩张

- 扩到 120-160 物种
- 图鉴、关系图、事件系统、任务和干预系统上线

## 3. 代码迁移原则

- 先包裹旧系统，再替换
- 先建新模块，再迁移旧逻辑
- 不再继续把所有功能堆进当前 `Ecosystem`

## 4. 每阶段验收

### 架构验收

- 是否多尺度
- 是否数据驱动
- 是否关系独立

### 生态验收

- 是否形成完整链条
- 是否存在级联变化
- 是否支持关键种影响

### 游戏验收

- 是否可视化
- 是否可干预
- 是否可形成任务和叙事

## 5. 实际开工顺序

1. 写入 v4 文档
2. 新建 `world` 与 `data` 模块
3. 定义 `SpeciesTemplate` 和 `RelationTable`
4. 让当前 `Ecosystem` 成为 `RegionSimulation` 前身
5. 先做温带森林区和湿地区
6. 优先落地关键种：狼、鹿、河狸、翠鸟、狗鱼、鲶鱼、青蛙、蝙蝠、蜜蜂
