# EcoWorld 总主线计划

本文档用于统一当前项目的总目标、阶段路线和前后端协作顺序。

它不是替代旧的 `v4` 文档，而是把下面三条线收成一张总路线图：

- `生态世界主线`：`world / region / ecology / species / long-term feedback`
- `可探索游戏主线`：Godot 世界图、区域入口、2.5D/3D 场景探索
- `内容扩张主线`：持续增加物种、热点、关系、图鉴和任务

## 1. 总目标

项目原始目标没有变：

`EcoWorld` 要做成一个可以被玩家走进去探索、观察、理解并长期管理的活生态世界。

这意味着最终产品必须同时成立 4 层：

1. `世界层`
   玩家能在世界地图上查看多个生态区、区域状态、区域连接和长期趋势。

2. `区域层`
   玩家能进入不同区域，看到地形、路径、入口、出口、热点和动物群落。

3. `生态层`
   区域内的物种、关系、链路、压力和长期反馈是真的在运行，而不是静态背景。

4. `玩法层`
   玩家不只是看场景，而是有探索目标、观察记录、图鉴进度、区域跳转和长期治理目标。

## 2. 产品形态

最终产品不是：

- 纯后台控制台
- 纯策略面板
- 纯单区域动物展示器
- 纯美术场景漫游

最终产品应该是：

`世界图导航 + 区域内 2.5D/3D 探索 + 活生态系统 + 图鉴观察/长期治理`

也就是说：

- `世界图`：做导航和世界态势，不承担主要游玩时长
- `区域场景`：承担主要探索和观察体验
- `后端生态系统`：持续驱动区域状态、动物分布、链路和长期变化
- `图鉴/观察/事件/任务`：把模拟结果转成玩家能理解和追踪的内容

## 3. 旧计划与新计划的关系

### 3.1 不变的部分

下面这些旧主线仍然有效，而且必须保留：

- `v4` 世界架构
- `WorldSimulation / RegionSimulation`
- `social / territory / grassland / carrion / wetland`
- 多区域并行
- 长期反馈回路
- 关键物种与生态链
- 未来扩充到更多物种和区域

### 3.2 需要调整的部分

需要调整的是“玩家入口形式”：

- 旧方向偏成了：`世界总控界面 -> 更多策略面板`
- 新方向应该是：`世界图导航 -> 进入区域 -> 在场景里探索和观察`

所以变化的是：

- `产品表达方式`
- `前端主界面结构`

而不是：

- 世界搭建目标
- 生态系统目标
- 长期内容扩张目标

## 4. 三条主线

后面所有开发都按 3 条主线协同推进，不再单独只推一条。

### A. 生态世界主线

负责“这个世界本身是否真实”：

- 世界区划
- 区域生态链
- 物种关系
- 长期压力
- 级联反馈
- 关键物种影响

对应代码层：

- `src/sim`
- `src/ecology`
- `src/entities`
- `src/world`
- `src/data`

### B. 可探索游戏主线

负责“玩家是否真的像在玩游戏，而不是看面板”：

- 世界图导航
- 区域入口
- 2.5D/3D 区域探索
- 主角移动
- 场景地形
- 动物偶遇
- 热点观察
- 区域跳转

对应前端层：

- `godot/scenes`
- `godot/scripts`
- `src/ui/world_payload.py`

### C. 内容扩张主线

负责“世界是否越来越丰富，而不是永远只有样板物种”：

- 新物种
- 新热点
- 新群落
- 新图鉴内容
- 新观察任务
- 新区域差异

这条线必须长期存在，不是收尾阶段才开始。

## 5. 总里程碑

后面统一按 8 个里程碑推进。

### M1 探索框架稳定化

目标：

先把“能玩”打稳。

前端：

- 修正控制方向
- 做真实碰撞与阻挡
- 扩大区域尺度
- 压缩首屏可见范围
- 稳定入口/出口切换流程
- 让路线/可走区/不可走区一眼可读

后端：

- 每区稳定导出：
  - `dominant_biomes`
  - `species_manifest`
  - `exploration_hotspots`
  - `frontier_links`
  - 稳定区域状态摘要

验收：

- 角色能稳定探索
- 不会“整图随便走”
- 不会“一眼看全图”
- 不同区域都能稳定进出

### M2 区域关卡化

目标：

让草原、湿地、森林、海岸真的像不同地图，而不是同一张图换颜色。

前端：

- 不同群系入口语言
- 不同路线组织
- 不同障碍结构
- 不同热点地标
- 不同入口三段式构图

后端：

- 每区增加更明确的：
  - 入口类型
  - 热点类型
  - 路线权重
  - 动物锚点分布

验收：

- 玩家切区后能一眼看出“这是另一个地方”
- 不是靠 HUD 文本区分区域

### M3 动物辨识与基础行为

目标：

玩家一眼能认出关键动物，而且知道它们是活的。

前端：

- 关键动物 silhouette 分化
- 草食群 / 掠食群 / 飞行群 / 水域生物外形拉开
- 主个体更明显
- 近距离惊动、回望、再聚群

后端：

- 每种动物输出更明确的：
  - 行为类型
  - 群体角色
  - 生态位

验收：

- 大象看得出是大象
- 狮子看得出是狮子
- 长颈鹿、斑马、河马、鳄鱼、秃鹫等至少能被区分

### M4 生态互动闭环

目标：

动物之间真的互相影响，不再只是同场景活动。

前端：

- 追逐/追猎
- 水源聚群
- 腐食响应
- 飞行 scavenger 聚集
- `leader / sentry / alpha` 带动整群重排

后端：

- 输出：
  - `pressure events`
  - `chase results`
  - `carrion response`
  - `regional behavior anchors`

验收：

- 玩家能直接观察到：
  - 草食群受压改线
  - 掠食者前压
  - 飞行动物对腐食区响应

### M5 观察与图鉴玩法

目标：

探索不只是走动，而是有明确记录和观察目标。

前端：

- 观察任务
- 热点驻留采样
- 动物偶遇记录
- 区域图鉴
- 发现日志

后端：

- 输出：
  - 任务目标
  - 完成状态
  - 发现列表
  - 区域生态摘要

验收：

- 玩家知道自己在看什么、记录什么、下一步该去哪里

### M6 世界图回接

目标：

把世界图从“旧主界面”改成“导航层”。

前端：

- 世界图负责：
  - 选大区
  - 看区域状态
  - 看区域连接
  - 进入区域
- 不再承载主要游玩时间

后端：

- 世界状态和区域探索结果双向联动

验收：

- 玩家在世界图上做导航，在区域里做探索

### M7 内容扩张

目标：

按“区域群落完整化”持续扩物种，而不是随机加动物。

扩法必须遵循：

1. 先补草原完整群落
2. 再补湿地完整群落
3. 再补森林完整群落
4. 再补海岸/浅海完整群落

每加一批物种，必须同时接入：

- 分布
- 行为
- 热点关系
- 图鉴
- 观察任务
- 区域生态链

### M8 长期治理与世界目标

目标：

把项目从“探索生态世界”推进到“长期管理生态世界”。

需要补：

- 章节目标
- 世界治理目标
- 区域退化/恢复目标
- 长期成败条件
- 决策前后变化表达

## 6. 实施顺序

后面严格按这个顺序推进：

1. `M1 探索框架稳定化`
2. `M2 区域关卡化`
3. `M3 动物辨识与基础行为`
4. `M4 生态互动闭环`
5. `M5 观察与图鉴玩法`
6. `M6 世界图回接`
7. `M7 内容扩张`
8. `M8 长期治理与世界目标`

不再跳着做。

## 7. 前后端协作规则

### 规则 1

前端新增玩法前，后端必须先能稳定输出对应 payload。

### 规则 2

后端新增复杂生态逻辑后，前端必须给出玩家可见的反馈，不允许长期只存在于数据层。

### 规则 3

新增物种必须同时接：

- 世界层
- 区域层
- 行为层
- 图鉴层
- 任务层

不允许“数据里有，玩法里没有”。

### 规则 4

世界图只做：

- 导航
- 世界态势
- 区域选择

主要玩法时长必须落在区域探索层。

## 8. 当前状态判断

当前项目最接近的阶段是：

- `M1`：进行中
- `M2`：已开始，但未稳定
- `M3`：已开始，但仍是原型

也就是说，现在最应该做的，不是继续堆世界层面板，也不是无脑加新物种，而是先把：

- 探索控制
- 场景尺度
- 碰撞与路线
- 区域差异
- 关键动物辨识

这些真正决定“能不能玩”的东西收稳。

## 9. 后续扩物种原则

后面一定会持续加生物，而且必须越来越丰富。

但扩法必须是：

`按区域群落完整化扩张`

而不是：

`按想起什么就随便多加几个`

正确做法：

- 草原：
  - 草食群
  - 掠食群
  - scavenger
  - 大型漫游者
- 湿地：
  - 水域动物
  - 两栖
  - 湿地鸟类
  - 近水大型动物
- 森林：
  - 林下草食
  - 伏击掠食者
  - 树栖/飞行生物
- 海岸/浅海：
  - 小型鱼群
  - 甲壳
  - 海岸鸟类
  - 浅海链条

## 10. 当前下一步

按这份总计划，当前下一步应该明确锁定为：

`继续完成 M1：探索框架稳定化`

具体就是：

1. 把控制、碰撞、路径、尺度先彻底收稳
2. 让区域不再一眼看完
3. 让关键动物能被认出来
4. 让基础生态互动可读

在 `M1` 没稳之前，不继续大规模扩面板和新系统。
